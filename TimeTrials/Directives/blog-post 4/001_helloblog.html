Angular Directives allow simplifying view structuring, decreasing wasted keystrokes and potential errors, as well as extensibility for transitioning the model into the view. In this first day of implementing Angular where I use to assist simplify the html-view.

Angular has gotten a lot of buzz lately being rumored to greatly reduce the lines of code required to make any site. Here I'll share my perspective on using it for reducing clutter in the view of this boostrap-based-blog.

Basically, I used a lot of ng-include and directives. Those familiar with php recall the way which require worked to include static blocks of code and simplify generation. This ng-include works in almost the same way with a crazy caveat: you will need to place extra tick marks '' around the html in order for it to work (trust me you need to include the extra '').

For example, the following line were used to bring in the content of this blog-entry:

 <div ng-include src="'001_helloblog.html'"></div>

Directives are another story. They are great for hiding away the usual html (also keeping it safe from stray keystrokes), while allowing us to access the key bits. Possibly not the best example, but here is how I am hiding away html code from my date time section:


app.directive("posttime",function() {
  return{
    restrict: "EA",
    scope: {
      dastime:"@"
    },
    template: " <p><span class=\"glyphicon glyphicon-time\"></span> Posted on {{dastime}}</p>"
  };
});

Including the custom html tag was simple and now looks like:

<posttime dastime=" Tue Oct 22 20:15:22 PDT 2013 "></posttime>

as opposed to:

<p><span class="glyphicon glyphicon-time"></span> Posted on August 24, 2013 at 9:00 PM</p>

Not tremendous gains, but one can easily image for places where dense code has only one value (navbars) which needs to be flipped to active (like tabs) this ability of directives can really be useful.
